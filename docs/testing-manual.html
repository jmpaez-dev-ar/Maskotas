🧪 Testing Manual - Formulario de Contacto Maskotas

📋 Instrucciones de Testing

Objetivo: Verificar que todas las funcionalidades del formulario de contacto funcionen correctamente.

Método: Sigue cada test paso a paso y marca como completado cuando funcione correctamente.

🔗 Abrir Sitio Web para Testing

🎯 Test 1: Validación en Tiempo Real

  • Ir al formulario de contacto (sección #contacto)
  • Escribir en el campo "Nombre" menos de 2 caracteres → debe mostrar error
  • Escribir un nombre válido → debe mostrar estado de éxito (verde)
  • Escribir email inválido (sin @) → debe mostrar error
  • Escribir email válido → debe mostrar estado de éxito
  • Escribir en mensaje menos de 10 caracteres → debe mostrar error
  • Los errores deben aparecer al salir del campo (blur)

💾 Test 2: Auto-guardado

  • Llenar parcialmente el formulario (nombre, email)
  • Esperar 2-3 segundos → debe aparecer indicador "Guardando borrador..."
  • Refrescar la página (F5)
  • Volver al formulario → los datos deben estar recuperados
  • Debe aparecer mensaje "Se recuperaron datos del formulario..."

⌨️ Test 3: Atajos de Teclado

  • Presionar Alt + C → debe navegar suavemente al formulario
  • Llenar el formulario completamente
  • Presionar Ctrl + Enter → debe enviar el formulario
  • Presionar Ctrl + Alt + R → debe preguntar si reiniciar
  • Usar Ctrl + ↑/↓ dentro del formulario → debe navegar entre campos

🧠 Test 4: Autocompletado Inteligente

  • Seleccionar "Consulta general" en asunto → mensaje debe auto-completarse
  • Seleccionar "Emergencia" → debe agregar texto de emergencia
  • Seleccionar "Cita veterinaria" → debe agregar texto de cita
  • El cursor debe posicionarse al final del texto
  • Solo debe funcionar si el campo mensaje está vacío

📊 Test 5: Indicador de Progreso

  • La barra de progreso debe aparecer arriba del botón enviar
  • Debe estar en 0% inicialmente (sin color)
  • Debe aumentar al completar campos requeridos
  • Debe cambiar de azul → naranja → verde según progreso
  • Debe llegar a 100% cuando todos los campos requeridos estén válidos

🔄 Test 6: Contador de Caracteres

  • Escribir en el campo mensaje → debe aparecer contador "X / 1000 caracteres"
  • Con más de 800 caracteres → debe volverse naranja (caution)
  • Con más de 900 caracteres → debe volverse rojo y parpadear (warning)
  • El contador debe actualizarse en tiempo real

🌐 Test 7: Simulación de Conectividad

  • Abrir DevTools → Application → Service Workers
  • Marcar "Offline" → debe aparecer mensaje de sin conexión
  • Desmarcar "Offline" → debe aparecer mensaje de conexión restaurada
  • Los mensajes deben ser informativos y desaparecer automáticamente

🔒 Test 8: Protección contra Pérdida

  • Llenar parcialmente el formulario
  • Intentar cerrar la pestaña → debe aparecer confirmación
  • Enviar formulario exitosamente
  • Intentar cerrar pestaña → NO debe aparecer confirmación

♿ Test 9: Accesibilidad

  • Navegar solo con Tab → todos los elementos deben ser accesibles
  • Los campos con error deben tener outline visible al recibir foco
  • Labels deben estar asociados correctamente con inputs
  • Mensajes de error deben ser anunciados por lectores de pantalla
  • Campos requeridos deben tener asterisco (*) visible

📱 Test 10: Responsividad

  • Cambiar a vista móvil (DevTools → Device Mode)
  • El formulario debe cambiar a una columna
  • Los botones deben ser más grandes en móvil
  • Los mensajes deben ajustarse al ancho de pantalla
  • Touch interactions deben funcionar correctamente

🎨 Test 11: Animaciones

  • Recargar página → los campos deben aparecer con animación escalonada
  • Hacer click en campos → debe haber efecto de foco suave
  • Mensajes de notificación → deben deslizarse desde la derecha
  • Botón de envío → debe mostrar spinner animado durante carga

💡 Test 12: Tips de Usabilidad

  • Primera visita → deben aparecer tips después de 5 segundos
  • Tips deben explicar atajos de teclado disponibles
  • Tips deben desaparecer automáticamente después de 4 segundos
  • En sesiones posteriores → tips no deben aparecer

✅ Test 13: Envío del Formulario

  • Llenar formulario completamente con datos válidos
  • Marcar checkbox de términos y condiciones
  • Hacer click en "Enviar mensaje"
  • Botón debe mostrar "Enviando..." con spinner
  • Después de 2 segundos → mensaje de éxito
  • Formulario debe limpiarse automáticamente
  • Datos de localStorage deben eliminarse
